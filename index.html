The League of Extraordinary Packages

View on GitHub

The League of Extraordinary Packages has one driving mission, to track down and develop PHP packages of the highest quality possible, using modern tools, standards and practices to distribute, test and strengthen the code.

Members of The League follow the standards put forward by the PHP-FIG, follow the best practices detailed by PHP The Right Way and distribute all code through Composer.

Packages

  • geotools - Geotools is a PHP geo-related library, built atop Geocoder and React libraries.
  • grunt - Grunt is PHP library for executing commands in parallel on multiple remote machines, via SSH. Simplify and automate deployment.
  • oauth1-client (WIP) - A client library for integrating with OAuth 1.0 providers.
  • oauth2-client - Make it stupidly simple to integrate your application with OAuth 2.0 identity providers.
  • oauth2-server - A standards compliant OAuth 2.0 authorization server and resource server.
  • monga - A simple and swift MongoDB abstraction layer.
  • statsd - A library for working with StatsD in PHP.
  • twitter (WIP) - A high quality Twitter API library, ported from python-twitter.

Rules

We accept any packages into the group as long as they solve a valid problem, and follow the following rules:

  1. Use PSR-2, with League\ as the PSR-0 namespace.
  2. List on Packagist, with the league\ vendor namespace.
  3. Shove code in a src/ folder.
  4. Write unit tests. Aim for at least 80% coverage for v1.0.
  5. DocBlock all the things.
  6. SemVer must be used to manage version numbers.
  7. An extensiive README.
    • Installation. Example
    • Usage. Example
    • Requirements
    • Contribution instructions (link to a CONTRIBUTING.md)
    • Framework Integration (Service Providers for Silex and/or Laravel 4 at minimum)
    • Author
    • License (MIT or DBAD)
    • Acknowledgements
  8. Include Travis-CI widget in README.
  9. Include Packagist widgets. Example.
  10. List Phil Sturgeon and Alex Bilbie as a maintainer on Packagist, to avoid "hit by a bus" scenarios.

Support

The League hangs out in #php-loep on Freenode IRC, so hopefully there will be a member or a user who can help.

If nobody is on IRC to help you, or you don't know how IRC works, throw up an issue on GitHub. Please do not email any of the League directly!. There are multiple developers per project, so emailing directly is not only a bad idea for you, but its rude because we've asked you not to. Issues get answered as soon as possible. League members are allowed holidays.

Finally, League members are not responsible for helping you integrate our packages with your favorite framework. You have three options for integration:

  1. Use your framework documentation or forums to find a simple integration method.
  2. Send a pull request to the package to add a Service Provider (Silex and Laravel 4 has these).
  3. Make (or find) a glue package, for example: "cakephp-monga" would be a simple package which lists "monga" as a dependency and wraps the API, or config, or whatever it needs to be CakePHP specific.
